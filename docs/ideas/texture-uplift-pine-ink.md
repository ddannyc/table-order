# 质感升级：首页 + 菜单（松墨 Pine-Ink）

> idea-refine 产出。借参考图（mp-ui / 四月春膳）的质感纪律，换松墨绿+金独立配色。
> 兼容既有 idea → `/plan` → `/build` 流水。基线：绝不 git add `frontend/config.js` / `.claude/` / `specs/`。

## Problem Statement
**HMW** 在不抛弃 weui 组件根基的前提下，给首页和菜单注入一套"松墨"温暖品牌识别（深松绿 + 燕麦奶油 + 琴酒金 + 线描插画），让它从"通用 weui 脚手架"变成有记忆点的产品——同时守住 a11y 与改动可回滚。

## Recommended Direction
借用参考图的**质感纪律**（一个深主色压场 + 暖中性背景 + 一个暖强调色 + 线描插画做"温暖载体"），换成松墨绿+金的独立配色，避开"奶油+衬线+陶土橙"这一 AI 默认套路。weui 官方支持用 CSS 变量覆盖换肤（与 `data-weui-theme="dark"` 同机制），所以**全部改动落在 `app.wxss` 令牌层 + 两个页面的 wxss/wxml，不 fork weui、不动组件结构**。signature 只花在一处：**单线 SVG 插画集**（首页 hero + 菜单分类占位 glyph + 未绑桌空状态），金色描边。

## Token 系统（落到 `app.wxss page{}`，覆盖 weui 变量）
```
主色   --weui-BRAND  #2C4A3B   品牌栏 / 主按钮 / 菜单激活竖条(已用此变量)
背景   --weui-BG-0   #F3EEE4   页面·燕麦奶油
卡面   --weui-BG-1   #FBF8F2   卡片/抬升(近白暖)
暖panel --weui-BG-2  #EFE9DD   门店栏 / 左类目轨底
文字   --weui-FG-0   #2A2723   暖近黑正文（12:1 ✓）
       --weui-FG-2   #6E665A   次要文字（4.89:1 ✓，比参考 #8A8275 调暗以过 AA）
描边   --weui-FG-3   #E3DCCE   暖发丝线
强调   --brand-accent #C98A2B  金：插画描边 / 分隔 / 印章（仅填充与描边，绝不做文字）
价格   --price-ink   #A66E1F   暗金铜：价格大字（3.73:1，仅 ≥34rpx 粗体）
```
**关键 a11y 取舍（已据实算 WCAG 自审）**：
- 亮金 `#C98A2B` 做文字对比过低 —— 只用于*填充/描边/插画*，绝不做文字。
- 徽章/数量 chip 用**深绿底 + 奶油字**（8.44:1），不用金底白字（2.94:1 不达标）。
- 价格用暗金铜 `#A66E1F`，必须大字粗体（3.73:1 过 3:1 大字阈值）；`起`后缀小号 → 用 `--weui-FG-2`。
- 次要文字令牌从参考 `#8A8275`(3.28:1 不达标) 调暗到 `#6E665A`(4.89:1)。
- 激活态不只靠颜色（绿竖条 + 加粗，沿用 M4）。

## Type（受小程序限制的诚实方案）
小程序自定义显示字体需 `loadFontFace`/打包字体（重）。不赌字体：沿用现有 `--font-*` 比例，靠字重/字号/字距拉层级；品牌字标用字距收紧 + `--font-h1`。signature 留给颜色与插画，不留给字体。

## Signature：单线 SVG 插画集（金色描边，inline，零二进制）
- **首页 hero**：单线场景横幅（替代参考图厨师插画位）。
- **菜单分类占位**：把 M3 中性文字占位块升级为每分类单线 glyph（cup/bubble/cheese/sparkle 已在 `menu-image.js` 枚举）——占位从"图没了"变成刻意品牌纹理。
- **未绑桌空状态**：单线空碗/纸飞机 + 行动指引。

## Layout（仅两屏，结构基本不动）
- **首页**：顶部深松绿品牌band（字标 + 余额）；下方 hero 线描横幅；堂食/外卖两入口卡换暖卡面 + 金发丝线 + 深绿标题。
- **菜单**：沿用 M4 左轨+大卡；门店栏→暖panel；激活竖条→真品牌绿；卡片→近白抬升 + 暖发丝线；价格→暗金铜；占位缩略图→金线 glyph；购物车结算条→深松绿。

## Key Assumptions to Validate
- [ ] 金/铜对比达标：模拟器实测 `#A66E1F` 价格、`#8A8275` 次要文字在 `#F3EEE4`/`#FBF8F2` 上达标。
- [ ] weui 变量覆盖无副作用：弹层/cell/按钮无对比或可读性回退。
- [ ] inline SVG 真机渲染：`<image>` data-URI / 背景图 哪种最稳，需真机验证。

## MVP Scope
**In**：`app.wxss` 令牌覆盖；首页 hero + 品牌band + 入口卡重皮；菜单重皮 + 占位 glyph 升级；1 个未绑桌空状态插画。
**Out**：见下。

## Not Doing（本期明确不做）
- 真实食物摄影 —— 选了线描插画；摄影是另一条供给链，且与 M3 零二进制决定冲突。
- 其余 4 屏（下单/我的订单/地址/选店）重皮 —— 先用两旗舰页验证令牌系统再铺开（可回滚）。
- 自定义衬线/显示字体 —— 小程序成本高、易拖慢；signature 由颜色+插画承担。
- 类目 scroll-spy / 锚点联动 —— 与质感无关。

## Resolved Decisions
- **hero 线描：具象·蒸笼一桌菜** —— 单线极简蒸笼+筷+碗场景，直接勾食欲，对齐参考图厨师插画的同意图。这是唯一 signature 时刻。
- **余额/会员积分不上首页** —— 属功能非质感，需后端（钱包/积分）支撑。本轮品牌 band 仅显字标，保持克制；余额/积分留作独立功能日后再做。
