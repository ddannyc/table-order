## 为什么改方向（诚实复盘）
上轮只换了 weui 配色、画了抽象单线、用了空占位块——参考图真正出彩的三样（彩色厨师插画、菜品实拍卡、陶土橙强调）一样没做，所以"没设计感"。这轮把这三样补齐。

## Problem Statement
**HMW** 把参考图的首页/菜品/下单/底部tab 如实复刻进小程序——墨绿+米白+陶土橙、彩色插画、照片优先菜品卡——只保留有真后端数据的模块，菜品图用展位图过渡、留 OSS 接口。

## 已锁定决策
- **菜品图** = 展位图（设计过的彩色矢量占位，非空块）过渡；卡片读 `product.image`，发布后填 OSS URL 即生效。
- **配色** = 完全照参考图（墨绿 + 米白 + 陶土橙）。金色退场。
- **功能保真** = 只复刻有真数据的：余额（`wallet.go`）、积分（`reward.go`）做；优惠券/自提/预约时间无后端，不做。
- **首页 IA** = 融合：墨绿品牌头 + 余额/积分真数据 + 厨师插画 banner + 堂食/外卖入口卡（参考图卡片样式）。保留单型在首页定的 IA。
- **Tab 图标** = 加墨绿 PNG 图标素材到 /static（图标本就该是图）。

## MVP Scope（四块 + 令牌）
- **令牌**：金→陶土橙，新增 --accent，price-ink 改深陶土；更新对比度测试。
- **首页（融合）**：墨绿品牌头（店名/品牌 + 余额 + 积分，真数据）→ 厨师彩色插画 banner → 堂食/外卖入口卡。
- **菜品选择**：左类目轨 + 类目数量徽章（真数据=品类菜品数）；右照片优先卡（大 thumb、展位图兜底、陶土橙价、绿色圆形加购）；绿色结算条。
- **下单**：店头 + 商品明细带照片缩略图 + 支付方式（余额支付 real + 微信支付）+ 合计/支付 + 支付成功态。
