# 餐饮门店扫码裂变小程序 PRD（基于视频分析）

## 视频核心玩法还原（基于视频逐帧分析）

视频展示的是一个典型的：

# “餐饮门店共享股东 / 消费返利裂变” 小程序

视频中的真实业务流程：

1. 顾客到店
2. 扫桌码进入小程序
3. 领取“股东资格/会员资格”
4. 消费后获得返利
5. 邀请好友继续消费
6. 用户持续获得奖励
7. 顾客从普通消费者变成“推广员”

视频中的核心营销话术：

- “只要扫码”
- “全自动裂变”
- “消费返10%”
- “不用发传单”
- “共享模式”
- “让客户帮你宣传”

视频里本质强调的是：

# 用户不是来领优惠券
# 而是被包装成“店铺股东”

这是整个产品转化率最高的关键点。

---

# 一、项目背景

视频核心逻辑：

用户在餐饮门店消费时，通过扫码登录小程序，获得一定比例的“消费返利/福利金”，并通过分享、邀请、复购等方式形成持续裂变。

视频中的核心卖点：

- 顾客扫码即可参与
- 消费后获得返利
- 用户会主动传播
- 商家无需持续投流推广
- 用“福利金/返现/会员体系”提高复购率

本质上是：

“餐饮会员 + 裂变分销 + 储值返利 + 社交传播”系统。

# 4. 福利金返利系统

## 功能说明

用户消费后获得一定比例福利金。

# 福利金直接进入小程序余额

但：

# 不允许提现
# 仅可用于后续消费抵扣

这是视频模式里非常关键的一点。

原因：

- 降低平台现金压力
- 避免金融风险
- 提高用户复购率
- 让资金沉淀在商家体系内

---

## 福利金使用规则

### 获得方式

- 自己消费返利
- 邀请好友消费奖励
- 活动奖励

---

### 使用限制

| 规则 | 示例 |
|---|---|
| 不可提现 | 禁止转微信零钱 |
| 不可转账 | 用户之间不可转赠 |
| 仅订单抵扣 | 下单时使用 |
| 抵扣比例限制 | 单笔最多抵扣30% |
| 有效期 | 90天 |

---

## 消费示例

用户余额福利金：

50元

本次订单：

100元

系统规则：

最多抵扣30%

则：

本次可抵扣30元。

用户实际支付：

70元。

真正让这个系统有效的核心不是“返利”。

# “让顾客觉得：不扫码就亏了。”

所以必须重点设计：

- 福利感
- 即时到账
- 裂变收益展示

# 十六、建议优先实现的版本

最推荐的切入方式：

# 不做平台
# 先做单店SaaS

原因：

- 风控简单
- 不涉及复杂清结算
- 商家更容易成交
- 可快速验证模型
- 现金流更稳定

第一版甚至可以：

- 不接支付
- 商家手动录单
- 手动审核提现

先验证：

“用户是否愿意裂变传播。”

验证成功后再扩展。
